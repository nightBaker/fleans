# Manual Test Plan — Self-host docs + Release runbook

> Verifies the docs shipped by #409: two new self-host guides
> (`guides/self-host-docker-compose.md`, `guides/self-host-helm.md`),
> the new "Self-host" sidebar group, and the `## Cutting a Release`
> CLAUDE.md runbook. Most steps are programmatic (build + grep);
> sidebar / theme rendering needs a browser.

## Scenario

The release pipeline at `.github/workflows/release.yml` produces two
consumer-facing artifacts on every `v<SemVer>` tag — `docker-compose-v<VER>.zip`
and `fleans-<VER>.tgz`. Until #409 ships, no docs tell users how to consume
these artifacts, and no runbook tells maintainers how to cut a release.

After #409 ships:
- Two new task-oriented guides under a new "Self-host" sidebar group walk users
  through release-asset download → install → upgrade → troubleshoot.
- The existing `reference/self-hosting.md` (Helm chart internals) and
  `reference/deployment.md` (non-Helm topology) remain canonical and are
  cross-linked from the new guides without content duplication.
- A new `## Cutting a Release` section in `src/Fleans/CLAUDE.md` documents
  the maintainer runbook with both pre-release dry-run sentinels and a
  correct `gh api`-based ghcr.io rollback path.

## Prerequisites

- `cd website && npm install` has been run at least once.
- Dev server NOT already running on port 4321.
- Branch `docs/409-self-host-runbook` (or main, post-merge) checked out.


**NOT** validated. If a future PR adds a link checker, expand this step
accordingly.
